## üîπ What is JavaScript?

JavaScript is a **synchronous** and **single-threaded** programming language.
This means it executes **one statement at a time**, in a **specific order**, from top to bottom.

Example:

```js
console.log("Hello JS...");
```

JavaScript will complete this line first, then move to the next line.

---

## üîπ What is an Execution Context?

An **Execution Context** is the environment in which JavaScript code is **created and executed**.

You can imagine it as a **big box** that manages:

* Variables
* Functions
* The execution of code

Every time JavaScript runs, it creates an execution context.

---

## üîπ Parts of an Execution Context

An execution context has **two main components**:

### 1Ô∏è‚É£ Variable Environment (Memory Component)

This is where JavaScript stores:

* Variables
* Functions
* Function arguments

Before the code starts running, JavaScript allocates memory for all variables and functions here.

Example:

```js
let a = 10;
function greet() {
  console.log("Hi");
}
```

In memory:

* `a ‚Üí undefined`
* `greet ‚Üí function()`

---

### 2Ô∏è‚É£ Thread of Execution (Code Component)

This is where the **actual code runs**.

JavaScript:

* Reads the code **line by line**
* Executes one line at a time
* Moves to the next line only after finishing the current one

Example:

```js
let a = 10;
console.log(a);
```

Steps:

1. Assigns `10` to `a`
2. Prints `10`

---

## üîπ How JavaScript Executes Code

JavaScript follows these steps:

### Step 1: Creation Phase

* Memory is allocated
* Variables are set to `undefined`
* Functions are stored fully

### Step 2: Execution Phase

* Code is executed line by line
* Values are assigned
* Functions are called

---

## üîπ Why JavaScript is Single-Threaded

JavaScript can do only **one task at a time** because it has:

* One call stack
* One main thread of execution

This makes JavaScript:

* Predictable
* Easier to manage
